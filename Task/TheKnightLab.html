The Knight - part 1

  • Example application
  • Resources
  • 4 points:
    • Change window's title
    • Starting position: center of the screen
    • The window has fixed size, but you can maximize it.
    • All controls update their size after maximization.
    • You should add menu bar with "File" button, which has submenus "New game", "Settings" and "Exit"
    • Board: 64 cells. Each cell has one of the following colors: ForestGreen or Maroon. Colors are choosen at random.
    • "New game" button creates new board. It has shortcut: ctrl+N.
    • "Exit" button closes the application.
    • Before closing the application confirmation window appears. When confirmation window is shown, the main window is inactive.
    • "Settings" button opens Settings window. Shortcut: ctrl+M
    • Settings window:
      • A new window which contains a combobox, which allows to change board's size (8x8,10x10,12x12). Combobox's default value: 8x8.
      • Settings window has also "OK" and "Cancel" buttons. After clicking "Cancel" changes are not taken into consideration.
      • Settings window cannot change size, it does not have maximize an minimize buttons.
      • Settings appears in the centre of the main window.
  • 8 points:
    • The knight image (image is loaded from resources).
    • You can move the knight using arrow keys.
    • After holding an arrow key, the knigt moves only by one cell
    • The knight rotates after changing direction (left/right).
    • The knight cannot move into walls (cells in Maroon color).
    • After the beginning of a new game or changing size of the board, the knight cannot be placed in the wall (it should appear on ForestGreen cell).
    • The knight background is transparent.
  • Hints:
    • Control.Dock, Control.Anchor
    • TableLayoutPanel
    • PictureBox
    • DialogResult
    • Bitmap.MakeTransparent
    • ImageLayout.Stretch
  • It is not possible to get points without fulfilling all requirements
  • The application should behave like the example
