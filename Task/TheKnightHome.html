The Knight - part 1

  • Example application
  • Resources
  • 4 points:
    • Change window's title
    • Starting position: center of the screen
    • The window has fixed size, but you can maximize it.
    • All controls update their size after maximization.
    • You should add menu bar with "File" button, which has submenus "New game", "Settings" and "Exit"
    • Board: 64 cells. Each cell has one of the following colors: ForestGreen or Maroon. Colors are choosen at random.
    • "New game" button creates new board. It has shortcut: ctrl+N.
    • "Exit" button closes the application.
    • Before closing the application confirmation window appears. When confirmation window is shown, the main window is inactive.
    • "Settings" button opens Settings window. Shortcut: ctrl+M
    • Settings window:
      • A new window which contains a combobox, which allows to change board's size (8x8,10x10,12x12). Combobox's default value: 8x8.
      • Settings window has also "OK" and "Cancel" buttons. After clicking "Cancel" changes are not taken into consideration.
      • Settings window cannot change size, it does not have maximize an minimize buttons.
      • Settings appears in the centre of the main window.
  • 8 points:
    • The knight image (image is loaded from resources).
    • You can move the knight using arrow keys.
    • After holding an arrow key, the knigt moves only by one cell
    • The knight rotates after changing direction (left/right).
    • The knight cannot move into walls (cells in Maroon color).
    • After the beginning of a new game or changing size of the board, the knight cannot be placed in the wall (it should appear on ForestGreen cell).
    • The knight background is transparent.
  • Hints:
    • Control.Dock, Control.Anchor
    • TableLayoutPanel
    • PictureBox
    • DialogResult
    • Bitmap.MakeTransparent
    • ImageLayout.Stretch
  • It is not possible to get points without fulfilling all requirements
  • The application should behave like the example

    • The Knight - part 2 (home)

    • Example application
    • Resources
    • Task decription:
      • On the menu bar only one window is shown.
      • Splash screen:
        • It appears right after starting the application (instead of the main window).
        • It disappears after 3s, then the main window appears.
        • It is round shaped and has a color different than default one.
        • It has label with: "The Knight".
        • You can move the window using mouse's left button. It is possible to move splash screen by clicking the label.
        • It appears and disappears gradually.
      • Game logic:
        • On the board there are also: key and closed door. After collecting the key, the door opens. The knight cannot enter closed door. After entering opened door, new board is generated.
        • The knight can destroy walls using space button. Only neighbouring walls are destroyed (without diagonal neighbours). It is not possible to destroy door or key.
        • Board generation: if a cell is neighbouring with a wall it has 50% chance to be a wall. If it does not have wall neighbours it has 20% chance to be a wall. Diagonal members are not taken into consideration.
      • Edit mode:
        • It is possible to enter edit mode by clicking "edit mode" button.
        • After entering edit mode, the button changes text to "Game mode", menu changes color and "Left click" button appears.
        • You can leave edit mode by clicking the same button again. The button changes text to "Edit mode", menu bar's color changes to default and "Left click" button disappears.
        • After clicking "Left click", "Grass" and "Wall" options appears. There should be a checkbox next to the option, which was choosen.
        • After clicking mouse's left button on a board's cell it changes to grass or wall, depending on which option is choosen in "Left click menu".
        • After clicking mouse's right button on a board's cell a context menu appears. It allows to place a knight, a key or closed door. On the map always must be one knight, one door and one key.
        • Int edit mode it is not possible to move knigt.
    • Hints:
      • Graphics.FillElipse, GraphicsPath, Region
      • Form.Opacity
      • ContextMenuStrip
